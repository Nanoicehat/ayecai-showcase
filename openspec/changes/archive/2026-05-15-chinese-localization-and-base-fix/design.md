## Context

现有 Vue 3 项目展示网站已搭建完成，但存在三个问题：
1. `vite.config.ts` 中 `base: '/demo-proj-260515/'` 导致所有 URL 包含仓库名
2. 界面全英文，不符合中文用户场景
3. demo 项目内容为英文占位文本，缺乏真实感

## Goals / Non-Goals

**Goals:**
- base 改为 `/`，URL 中不再出现仓库文件夹名
- 全站中文化（标题、导航、文案、项目内容）
- 三个现有 demo 项目内容中文化
- 新增一个 mock 项目，展示实际可用内容

**Non-Goals:**
- 不做 i18n 框架（不需要多语言切换）
- 不修改布局或样式系统
- 不改变项目架构

## Decisions

### 1. Base 路径直接改为 `/`

直接修改 `vite.config.ts` 中 `base` 为 `'/'`。部署到 GitHub Pages 时，如果是 `<user>.github.io` 仓库或使用自定义域名，base 为 `/` 即可。如果未来需要部署到子路径，可通过环境变量配置。

### 2. 硬编码中文文案

因为不需要多语言支持，所有中文文案直接写在组件中。不引入 i18n 库。

### 3. Mock 项目选择

新增第四个 mock 项目 `ai-chat-app`（AI 聊天助手），展示一个聊天界面风格的页面，使用渐变紫色主题。选择这个主题因为它与现有三个项目风格差异足够大。

## Risks / Trade-offs

- **[部署路径]** base 改为 `/` 后，如果部署到 `github.io/<repo>/` 子路径会导致资源 404 → 可通过 `VITE_BASE` 环境变量动态配置 base，但当前不做，保持简单
