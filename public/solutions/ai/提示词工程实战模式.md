## 模式一：角色设定 + 约束边界

```
你是一位资深的 Python 后端架构师，专注于高并发系统设计。
约束：只讨论技术方案，不涉及具体业务逻辑。
输出格式：方案名称 → 优缺点 → 适用场景。
```

## 模式二：少样本结构化输出

通过 2-3 个示例锚定输出格式：

| 输入 | 期望输出 |
|:--|:--|
| "用户登录失败" | `{severity: "medium", category: "auth", action: "check_credentials"}` |
| "服务器内存溢出" | `{severity: "critical", category: "infra", action: "scale_memory"}` |

## 模式三：思维链（Chain of Thought）

```
请一步步分析这个问题：
1. 首先确认问题的根本原因
2. 列出可能的解决方案
3. 评估每个方案的优缺点
4. 给出最终推荐
```

## 模式四：自洽验证（Self-Consistency）

让模型生成多个答案，取一致性最高的结果：

```
请用三种不同的思路解答这个问题，然后比较三个答案，给出最可靠的结论。
```

## 模式五：推理行动交替（ReAct）

```
思考: 我需要先查看用户的订单状态
行动: query_order(user_id="12345")
观察: 订单状态为"已发货"
思考: 用户可能想知道物流信息
行动: query_logistics(order_id="ORD-789")
```

## 模式六：思维树（Tree of Thought）

适合需要探索多条路径的复杂问题。每个分支独立评估后选最优路径。

## 模式七：提示链（Prompt Chaining）

将复杂任务拆成多个子提示词，前一个的输出作为后一个的输入。

## 模式八：元提示（Meta Prompting）

让模型先生成最优提示词，再用该提示词执行任务。

## 效果对比

| 模式 | 适用场景 | 成本 | 质量提升 |
|:--|:--|:--|:--|
| 角色设定 | 通用 | 低 | +15% |
| 少样本 | 格式敏感 | 中 | +30% |
| 思维链 | 推理任务 | 中 | +40% |
| 自洽验证 | 高可靠性 | 高 | +25% |
| ReAct | Agent | 高 | +50% |
